\section{Technical Overview}

The proof system is easiest to read as a structured certification procedure for
quantized GBDT training. The central challenge is not tree inference itself,
but proving that each round derives the correct per-node split statistics and
commits a split that optimizes the training objective. Two technical issues
recur throughout this process. First, the proof naturally produces claims over
different polynomial domains and tensor layouts. Second, the standard
split-search objective is usually implemented through sorting or feature scans,
which are awkward proof targets. Our overview therefore focuses on three
questions: what the committed forest must support, how the split objective is
rewritten into histogram statistics, and which batching tools let the resulting
checks compose succinctly.

\subsection{Forest Representation and Batched Inference}

We encode the ensemble using the hyperbox representation of \cite{cq++24}. Each
node carries a feature-wise half-open interval box, and a sample reaches a leaf
exactly when its quantized feature vector lies inside that leaf's box. The
committed forest contains:
\begin{itemize}
  \item routing matrices
  $\mathbf{L},\mathbf{R}\in\{0,1\}^{N_{\mathrm{tot}}\times N_{\mathrm{tot}}}$;
  \item lower and upper node bounds
  $\mathbf{N}^{\leftarrow},\mathbf{N}^{\rightarrow}\in[B+1]^{N_{\mathrm{tot}}\times d}$;
  \item a feature-selection matrix
  $\mathbf{E}\in\{0,1\}^{N_{\mathrm{tot}}\times d}$;
  \item a threshold tensor
  $\boldsymbol{\Theta}\in\{0,1\}^{N_{\mathrm{tot}}\times d\times B}$;
  \item a tree-index vector $\mathbf{tree}\in[TK]^{N_{\mathrm{tot}}}$ and a
  root matrix $\mathbf{Root}\in\{0,1\}^{TK\times N_{\mathrm{tot}}}$;
  \item a leaf-score vector $\mathbf{score}\in\mathbb{F}^{N_{\mathrm{tot}}}$.
\end{itemize}

This encoding supports both inference and training. For inference, the prover
fetches, for each tree-sample pair, the tree identifier, the lower and upper
box bounds, and the score stored at the accepting leaf, and then proves that
the replicated input lies inside the fetched box. Because the same input batch
is evaluated by all $TK$ trees, the protocol repeatedly needs to reinterpret one
logical batch under larger shapes and then aggregate the fetched scores back
into class scores.

\subsection{Training Proof Pipeline}

The training proof reuses the same committed forest but organizes the remaining
steps around a small sequence of relations. For each round $t\in[T]$ and class
$k\in[K]$, the prover obtains a global leaf-assignment vector
\[
\boldsymbol{\ell}^{(t,k)} \in
\{N_{\mathrm{int}},\ldots,N_{\mathrm{tot}}-1\}^{M},
\]
where $\boldsymbol{\ell}^{(t,k)}[r]$ is the global leaf index reached by sample
$r$ in the tree trained for class $k$ at round $t$.

The proof for one round then proceeds as follows.
\begin{enumerate}
  \item Update the cumulative score matrix and compute the pseudo-residuals.
  \item Aggregate leaf-indexed sample contributions into count and weighted
  histograms.
  \item Propagate those histograms from leaves to internal nodes through the
  committed routing matrices.
  \item Derive prefix statistics and total statistics, and then compute the
  candidate split score for every feature-threshold pair.
  \item Prove that the split recorded in $\mathbf{E}$ and $\boldsymbol{\Theta}$
  is a maximizer of the candidate score tensor.
  \item Aggregate residuals at the leaves to form the next round's leaf values
  and add them back into the score matrix.
\end{enumerate}

The tree-specific novelty lies in the middle of this pipeline: the protocol does
not try to prove a sorting procedure, but instead proves the statistics from
which split search can be reconstructed algebraically.

\subsection{Objective Rewriting for Split Search}

Consider one node $u$, one feature $j$, and one threshold bin $b$. Let
$I_u \subseteq [M]$ be the samples reaching node $u$, and let
$L_{u,j,b},R_{u,j,b}$ be the induced left and right partitions. If each sample
$r\in I_u$ carries target value $g_r$, then the residual-sum-of-squares
objective is
\[
\operatorname{RSS}(u,j,b)
=
\sum_{r\in L_{u,j,b}}(g_r-\bar{g}_L)^2
+
\sum_{r\in R_{u,j,b}}(g_r-\bar{g}_R)^2.
\]
Expanding the squares gives
\begin{equation}\label{eq:rss-rewrite}
\operatorname{RSS}(u,j,b)
=
\sum_{r\in I_u} g_r^2
-
\left(
\frac{\left(\sum_{r\in L_{u,j,b}} g_r\right)^2}{|L_{u,j,b}|}
+
\frac{\left(\sum_{r\in R_{u,j,b}} g_r\right)^2}{|R_{u,j,b}|}
\right).
\end{equation}
For fixed node $u$, the first term in \eqref{eq:rss-rewrite} is independent of
the candidate split. Therefore minimizing $\operatorname{RSS}(u,j,b)$ is
equivalent to maximizing
\begin{equation}\label{eq:split-objective}
\Phi(u,j,b)
:=
\frac{\left(\sum_{r\in L_{u,j,b}} g_r\right)^2}{|L_{u,j,b}|}
+
\frac{\left(\sum_{r\in R_{u,j,b}} g_r\right)^2}{|R_{u,j,b}|},
\end{equation}
subject to the validity condition that both children are nonempty.

This rewriting explains why histograms are sufficient proof objects. Once the
prover has committed, for every node $u$, feature $j$, and bin $q$, the sample
count and residual sum associated with that bin, the quantities
$|L_{u,j,b}|$, $|R_{u,j,b}|$,
$\sum_{r\in L_{u,j,b}} g_r$, and $\sum_{r\in R_{u,j,b}} g_r$ become prefix- and
total-sum expressions over the bin axis. The split objective is then reduced to
tensor algebra over committed counts and sums.

\subsection{Domain-Lifted and Interleaved Batching}

The construction repeatedly needs to batch claims that are structurally similar
but do not initially share one proof shape.

\paragraph{Domain-lifted batch.}
Suppose linear identities are expressed as divisibility claims
\[
F_i(X)=Q_i(X)\nu_{n_i}(X),
\qquad i\in[m],
\]
where the domain sizes $n_i$ may differ. Let
$N=\max_i n_i$ and define
\[
\widehat{F}_i(X):=F_i(X)\frac{\nu_N(X)}{\nu_{n_i}(X)}.
\]
Then every original claim holds if and only if every
$\widehat{F}_i(X)$ is divisible by $\nu_N(X)$, so the verifier can batch them as
\[
\sum_{i=1}^{m}\eta^{i-1}\widehat{F}_i(X)=Q(X)\nu_N(X)
\]
for a random challenge $\eta$. This is the right tool for heterogeneous linear
relations such as routing identities, padding-induced tensor equalities, and
batched zero checks over histogram-derived objects.

\paragraph{Interleaving batch.}
Now suppose several nonlinear checks share one pointwise shape
\[
\Psi\!\left(
f_i^{(1)}[x],\ldots,f_i^{(s)}[x]
\right)=0,
\qquad \forall x\in[n_i].
\]
If their domains differ, we first align them to a common size; in practice this
alignment is often achieved by replication. Once aligned, we interleave the
witness bundles coordinate-wise and obtain one larger bundle on which the same
pointwise relation $\Psi$ is checked once. This pattern is useful when the
protocol must prove several lookup-style table-difference checks or several
same-form rounding relations, because it avoids introducing one separate batch
for each local nonlinear subroutine.

The construction will package these two ideas as two reusable batching theorems:
one for heterogeneous linear claims and one for same-form nonlinear claims.
