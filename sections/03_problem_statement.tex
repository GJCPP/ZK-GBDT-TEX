\section{Technical Overview}

\subsection{Forest Representation}

We encode the trained ensemble using the hyperbox representation of
\cite{cq++24}. In this representation, each node is associated with a
feature-wise interval box, and a sample reaches a leaf exactly when its
quantized feature vector lies inside that leaf's box. Each tree is represented
by sparse left-child and right-child matrices together with node metadata. Let
$N_{\mathrm{tot}}$ denote the total number of committed nodes across all
trees, let $N_{\mathrm{int}}$ denote the number of internal nodes, and let
$N_{\mathrm{ext}}$ denote the number of leaves. Concretely, the committed forest
contains:
\begin{itemize}
  \item routing matrices $\mathbf{L},\mathbf{R}\in
  \{0,1\}^{N_{\mathrm{tot}}\times N_{\mathrm{tot}}}$;
  \item lower and upper bounds
  $\mathbf{N}^{\leftarrow},\mathbf{N}^{\rightarrow}\in[B+1]^{N_{\mathrm{tot}}\times d}$;
  \item a feature-selection matrix
  $\mathbf{E}\in\{0,1\}^{N_{\mathrm{tot}}\times d}$;
  \item a threshold tensor
  $\boldsymbol{\Theta}\in\{0,1\}^{N_{\mathrm{tot}}\times d\times B}$;
  \item a tree-index vector $\mathbf{tree}\in[TK]^{N_{\mathrm{tot}}}$ and a
  root matrix $\mathbf{Root}\in\{0,1\}^{TK\times N_{\mathrm{tot}}}$;
  \item a leaf-score vector $\mathbf{score}\in\mathbb{F}^{N_{\mathrm{tot}}}$.
\end{itemize}

This representation is well suited to both inference and training. Inference can
be reduced to fetching the unique leaf whose hyperbox contains the input, while
training can reuse the same routing information to determine which samples reach
which leaves and internal nodes.

\subsection{Inference Overview}

For a query batch $\mathbf{X}\in[B]^{b\times d}$, the prover first computes the
accepting leaf of every sample in every tree. It then proves two facts. First,
using matrix lookup, it fetches from the committed forest the lower and upper
hyperbox bounds and the score stored at the selected leaf. Second, it proves the
range check showing that the fetched hyperbox indeed contains the query input.

Because the same query batch is evaluated on all $TK$ trees, the protocol
conceptually replicates $\mathbf{X}$ once per tree and then aggregates the
fetched leaf scores along the tree dimension. This produces a class-score vector
for each sample. The only tree-specific complexity is the lookup into the forest
encoding; after that, the remaining work is tensor reshaping and summation.

\subsection{Training Overview}

The training proof reuses the same inference machinery on the training set. For
each round $t\in[T]$ and class $k\in[K]$, the prover obtains a leaf-index vector
\[
\boldsymbol{\ell}^{(t,k)} \in [N_{\mathrm{ext}}]^M,
\]
where $\boldsymbol{\ell}^{(t,k)}[r]$ is the leaf reached by sample $r$ in the
tree trained for class $k$ at round $t$.

\paragraph{Pseudo-residuals.}
Given the current score matrix $\mathbf{S}^{(t)}$, the prover computes the
pseudo-residuals $\mathbf{G}^{(t)}$. The exact softmax formula is deferred to
\cref{sec:construction} because it is not the tree-specific part of the proof.
At the overview level, what matters is that $\mathbf{G}^{(t)}[r,k]$ is the
regression target used to train the next tree for class $k$.

\paragraph{Leaf histograms.}
Recall from \cref{eq:rss-plain} that plaintext training evaluates a candidate
split by scanning thresholds after sorting samples by feature value, or
equivalently, after aggregating them into feature histograms. The core idea of
our proof is to use the second view. For each class $k$, node index $u$,
feature $j$, and bin $b$, the prover constructs two tensors:
\begin{align}
\mathbf{C}^{(t,k)}[u,j,b]
&=
\#\{r : \boldsymbol{\ell}^{(t,k)}[r]=u \text{ and } \mathbf{D}[r,j]=b\}, \\
\mathbf{H}^{(t,k)}[u,j,b]
&=
\sum_{r:\boldsymbol{\ell}^{(t,k)}[r]=u,\ \mathbf{D}[r,j]=b}
\mathbf{G}^{(t)}[r,k].
\end{align}
Here $\mathbf{C}^{(t,k)}$ counts how many samples fall into each
leaf-feature-bin triple, while $\mathbf{H}^{(t,k)}$ stores the corresponding sum
of residual targets. We refer to these tensors as count and weighted
histograms.

\paragraph{Propagation to internal nodes.}
Once leaf histograms are available, histograms for internal nodes are obtained
by summing the histograms of their children. This uses the committed routing
matrices $\mathbf{L}$ and $\mathbf{R}$ and avoids replaying branching logic from
scratch for every node.

\paragraph{From histograms to split scores.}
For a fixed node $u$ and feature $j$, prefix sums along the bin axis give the
statistics of sending all samples with feature value at most $b$ to the left
child:
\[
\mathbf{C}^{(t,k)}_{\leq}[u,j,b]
=
\sum_{q=0}^{b}\mathbf{C}^{(t,k)}[u,j,q],
\qquad
\mathbf{H}^{(t,k)}_{\leq}[u,j,b]
=
\sum_{q=0}^{b}\mathbf{H}^{(t,k)}[u,j,q].
\]
Subtracting from the totals yields the corresponding right-child statistics
$\mathbf{C}^{(t,k)}_{>}$ and $\mathbf{H}^{(t,k)}_{>}$. With these tensors, the
regression-tree objective can be rewritten using only counts and sums; no sample
sorting is needed at proof time.

\paragraph{Selecting the committed split.}
After computing the candidate score for every feature-threshold pair, the prover
must show that the split recorded in $\mathbf{E}$ and $\boldsymbol{\Theta}$ is a
maximizer. This is the final training-specific check: the committed tree
structure must match the optimizer of the split objective implied by the
committed histograms.

\subsection{Why Histogram Matters}

This histogram view is the main difference between our proof architecture and a
generic-circuit treatment of GBDT training. The expensive combinatorial part of
training is no longer ``prove the per-feature sorting step in zero knowledge.''
Instead, the
proof reduces split search to four structured tasks:
\begin{enumerate}
  \item prove correct leaf assignment for each sample;
  \item aggregate samples into per-bin histograms;
  \item derive prefix statistics and candidate split scores;
  \item prove that the committed split is the maximizer.
\end{enumerate}

The rest of the paper turns this overview into concrete polynomial identities
and commitment checks.
