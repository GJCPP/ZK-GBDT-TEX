\section{Preliminaries}

\subsection{Notation and Polynomial Encoding}

We work over a finite field $\mathbb{F}$. All vectors, matrices, and tensors are
flattened in row-major order before commitment. For $n \in \mathbb{N}$, we write
$[n] := \{0,1,\ldots,n-1\}$.

All committed lengths are assumed to be powers of two. To make this precise, we
fix an integer $L \ge 1$ such that every committed object is smaller than $2^L$,
and is padded to a length
dividing $2^L$. We also fix a primitive $2^L$-th root of unity $\omega \in
\mathbb{F}$. For every power of two $n$ with $n \mid 2^L$, we define
\[
\omega_n := \omega^{2^L/n}
\qquad\text{and}\qquad
\mathbb{H}_n := \langle \omega_n \rangle.
\]
Thus $\mathbb{H}_n$ is the multiplicative subgroup of size $n$ used as the
evaluation domain for length-$n$ vectors. Its vanishing polynomial is
\[
\nu_n(X) := \prod_{a \in \mathbb{H}_n}(X-a) = X^n-1.
\]
Because all domains are derived from the same root $\omega$, we have
$\omega_{nm}^m=\omega_n$ whenever $n$, $m$, and $nm$ are powers of two dividing
$2^L$. This compatibility is used later when we align objects of different
shapes.

Let $\lambda_n^{(i)}(X)$ denote the $i$-th Lagrange basis polynomial on
$\mathbb{H}_n$.

For a vector $\mathbf{v}\in\mathbb{F}^n$, its polynomial encoding over
$\mathbb{H}_n$ is
\[
v(X) := \sum_{i \in [n]} \mathbf{v}[i] \lambda_n^{(i)}(X),
\]
where $\mathbf{v}[i]$ denotes the $i$-th entry of $\mathbf{v}$.
If $\mathbf{V}$ is a matrix or higher-order tensor, we use $\mathbf{V}[i, j, k,\dots]$.

Throughout the paper, we index all nodes globally across all trees. We write
$[N_{\mathrm{int}}]$ for indexing internal nodes and
$\{N_{\mathrm{int}},\ldots,N_{\mathrm{tot}}-1\}$ for leaves.

\subsection{KZG Polynomial Commitments}

We use the KZG polynomial commitment scheme~\cite{kate2010constant}. Let
$\mathbb{F}$ be a field, and let $\mathcal{P}_{<d}$ denote the set of univariate
polynomials over $\mathbb{F}$ of degree strictly smaller than $d$.
A KZG commitment scheme for degree bound $d_{\max}$ consists of the following
algorithms.

\begin{itemize}
    \item $\Setup(1^\lambda, d_{\max}) \to \mathsf{pp}$:
    on input the security parameter $\lambda$ and a maximum supported degree
    $d_{\max}$, outputs public parameters $\mathsf{pp}$.
    In the structured-reference-string setting, $\mathsf{pp}$ contains the KZG
    commitment key and verification key for polynomials of degree $< d_{\max}$.

    \item $\Commit(\mathsf{pp}, f) \to C$:
    on input $\mathsf{pp}$ and a polynomial $f \in \mathcal{P}_{<d_{\max}}$,
    outputs a commitment $C$ to $f$.

    \item $\Open(\mathsf{pp}, f, z) \to (v,\pi)$:
    on input $\mathsf{pp}$, a polynomial $f \in \mathcal{P}_{<d_{\max}}$, and a point
    $z \in \mathbb{F}$, outputs the evaluation $v = f(z)$ together with an opening
    proof $\pi$ attesting that the committed polynomial evaluates to $v$ at $z$.

    \item $\Verify(\mathsf{pp}, C, z, v, \pi) \to \{0,1\}$:
    on input $\mathsf{pp}$, a commitment $C$, a point $z \in \mathbb{F}$, a value
    $v \in \mathbb{F}$, and an opening proof $\pi$, outputs $1$ iff $\pi$
    is a valid proof that the polynomial committed in $C$ evaluates to $v$ at $z$.

    \item $\ProveDegBound(\mathsf{pp}, f, d) \to \pi_{\deg}$:
    on input $\mathsf{pp}$, a polynomial $f \in \mathcal{P}_{<d_{\max}}$, and a target
    degree bound $d \le d_{\max}$, outputs a proof $\pi_{\deg}$ that
    $\deg(f) < d$.

    \item $\VerifyDegBound(\mathsf{pp}, C, d, \pi_{\deg}) \to \{0,1\}$:
    on input $\mathsf{pp}$, a commitment $C$, a target degree bound $d$,
    and a degree-bound proof $\pi_{\deg}$, outputs $1$ iff $\pi_{\deg}$
    is a valid proof that the polynomial committed in $C$ has degree $< d$.
\end{itemize}

We rely on the following standard properties of the commitment scheme.

\begin{itemize}
    \item \textbf{Correctness.}
    If a prover commits to a polynomial $f$ and then honestly generates an opening proof
    for a point $z$, verification accepts and returns the correct value $f(z)$.
    Likewise, if the prover honestly generates a degree-bound proof for $f$,
    then degree verification accepts.

    \item \textbf{Binding.}
    After publishing a commitment $C$, the prover cannot later open it to two different
    values at the same point. Intuitively, the commitment uniquely fixes the polynomial
    being committed to.

    \item \textbf{Sound degree certification.}
    A prover cannot produce a valid degree-bound proof for a polynomial whose degree
    exceeds the claimed bound, except with negligible probability.
\end{itemize}

\subsection{Gradient Boosted Decision Trees}

We prove the correct execution of a \emph{fixed quantized trainer semantics}.
It is closest to Friedman-style gradient boosting with regression trees on
pseudo-residuals, while borrowing the histogram-based split search familiar
from quantized systems such as XGBoost \cite{friedman01,xgboost16}. The paper
does \emph{not} claim to prove the full semantics of a production XGBoost
trainer with Hessian-based gains, regularization, or implementation-dependent
heuristics.

Let $\mathbf{D}\in[B]^{M\times d}$ be the quantized training dataset, where
$M$ is the number of samples, $d$ is the number of features, and each feature
value lies in one of $B$ bins. Let $\mathbf{Y}\in\{0,1\}^{M\times K}$ be the
one-hot label matrix for a $K$-class task. We treat the number of rounds $T$,
the learning rate $\mu$, the quantization parameters, and the tree-shape policy
as public parameters of the statement being proved.

Training proceeds for $T$ boosting rounds with public learning rate $\mu$. For
each round $t\in[T]$, the learner maintains a score matrix
$\mathbf{S}^{(t)} \in \mathbb{F}^{M\times K}$, where
$\mathbf{S}^{(t)}[r,k]$ is the current class-$k$ score of sample $r$ before the
$t$-th round is added. We initialize $\mathbf{S}^{(0)}=\mathbf{0}$. The class
probabilities are obtained by a row-wise softmax:
\begin{align}
\mathbf{P}^{(t)}[r,k] &:= \exp(\mathbf{S}^{(t)}[r,k]), \\
\mathbf{Z}^{(t)}[r] &:= \sum_{c=0}^{K-1}\mathbf{P}^{(t)}[r,c], \\
\boldsymbol{\pi}^{(t)}[r,k] &:= \frac{\mathbf{P}^{(t)}[r,k]}{\mathbf{Z}^{(t)}[r]}.
\end{align}
For the multinomial logistic loss, the pseudo-residual matrix is
\begin{equation}\label{eq:plain-pseudoresidual}
\mathbf{G}^{(t)}[r,k]
:=
\mathbf{Y}[r,k]-\boldsymbol{\pi}^{(t)}[r,k].
\end{equation}
Thus each column $\mathbf{G}^{(t)}[\cdot,k]$ is the regression target used to
train the tree for class $k$ in round $t$.

More concretely, for every class $k\in[K]$, the learner builds one regression
tree $f^{(t,k)}:[B]^d\to\mathbb{F}$ whose leaves carry scalar outputs. Once all
$K$ trees of round $t$ have been trained, the score matrix is updated by
\begin{equation}\label{eq:plain-score-update}
\mathbf{S}^{(t+1)}[r,k]
=
\mathbf{S}^{(t)}[r,k] + \mu\, f^{(t,k)}(\mathbf{D}[r,\cdot]).
\end{equation}
At inference time, the same rule is applied to a fresh sample
$\mathbf{x}\in[B]^d$: the ensemble accumulates the class scores
\[
\sum_{t=0}^{T-1}\mu\, f^{(t,k)}(\mathbf{x}),
\]
and the final class probabilities are obtained by applying softmax to those
$K$ accumulated scores.

The proof-critical step is therefore the training of each regression tree. A
tree is grown according to a fixed public shape policy; the proof then
certifies that the committed split metadata and leaf values are compatible with
that policy. At a node $u$, let $I_u \subseteq [M]$ denote the training samples
that currently reach $u$, and let
$g_r := \mathbf{G}^{(t)}[r,k]$ be the target value attached to sample $r$ for
the class being trained. For a feature $j$ and threshold bin $b$, splitting on
$(j,b)$ partitions $I_u$ into
\begin{align*}
L_{u,j,b} &:= \{r\in I_u : \mathbf{D}[r,j] < b\}, \\
R_{u,j,b} &:= \{r\in I_u : \mathbf{D}[r,j] \ge b\}.
\end{align*}
The standard residual-sum-of-squares objective is then
\begin{equation}\label{eq:rss-plain}
\operatorname{RSS}(u,j,b)
=
\sum_{r \in L_{u,j,b}} \left(g_r-\bar{g}_{L}\right)^2
+
\sum_{r \in R_{u,j,b}} \left(g_r-\bar{g}_{R}\right)^2,
\end{equation}
where $\bar{g}_{L}$ and $\bar{g}_{R}$ are the empirical means on the two sides.
The trainer selects the feature-threshold pair that minimizes
\eqref{eq:rss-plain} among \emph{valid} splits, namely those with
$L_{u,j,b}\neq\varnothing$ and $R_{u,j,b}\neq\varnothing$. The semantics proved
later does not attempt to canonicalize ties: any maximizing valid split is
accepted. If all candidates at a node are invalid, the masked objective is zero
for every candidate and the proved semantics is correspondingly nondeterministic
at that node. Once the structure is fixed, each leaf $\ell$ receives the
mean target value of the samples routed to that leaf:
\begin{equation}\label{eq:plain-leaf-value}
w_{\ell}^{(t,k)}
:=
\frac{1}{|I_{\ell}|}\sum_{r\in I_{\ell}} \mathbf{G}^{(t)}[r,k].
\end{equation}
In the proof, we later rewrite the split objective into a form that
depends only on per-bin counts and sums, because that representation is much
more amenable to polynomial commitments.

In standard plaintext training, this optimization is usually implemented by
sorting the samples at a node according to one feature at a time and then
scanning all candidate thresholds in that sorted order. In quantized systems
such as XGBoost, the same scan is often expressed through per-bin histograms
rather than explicit sorting on raw feature values. This is the gap we exploit:
instead of proving a sorting procedure in zero knowledge, we prove the
correctness of the histogram statistics from which the trainer derives the same
split scores.

\subsection{Quantization and Fixed-Point Reduction}

The proof works over $\mathbb{F}$ and does not emulate floating-point
arithmetic directly. Feature values already appear as public bin indices in
$[B]$. All remaining real-valued quantities are represented in fixed-point
form with one public scale
\[
\Delta := 2^q.
\]
For a real value $x$, the stored integer is $\lfloor \Delta x \rfloor$; decoding
divides by the same public scale. Signed values are first interpreted in the
intended signed-integer range and only then embedded into $\mathbb{F}$.

With this convention, addition and subtraction are scale-preserving. If
$\widehat{x}=\lfloor \Delta x\rfloor$ and
$\widehat{y}=\lfloor \Delta y\rfloor$, then
$\widehat{x}\pm\widehat{y}$ already represents the fixed-point value
$x\pm y$ at the same scale $\Delta$, so no explicit rescaling step is needed.

Multiplication and division are different. The product
$\widehat{x}\widehat{y}$ lives at scale $\Delta^2$, not at scale $\Delta$; and
an exact quotient generally leaves the fixed-point grid. Therefore every
operation that is supposed to return to the standard scale must be accompanied
by one explicit rounding or reduction step. Algebraically, the recurring check
is
\[
a = \left\lfloor \frac{b}{\Delta} \right\rfloor
\qquad\Longleftrightarrow\qquad
b = a\Delta + r
\quad\text{with}\quad
0 \le r < \Delta.
\]
This is the recurring fixed-point reduction relation used later in the
protocol: raw products or quotients are formed in the larger integer domain,
then reduced back to the standard scale by proving the above decomposition.
Because many such reductions share the same public divisor $\Delta$, they also
form a canonical family of same-form local checks for interleaving batching.

\subsection{Field Embedding and Range Assumptions}

All arithmetic is carried out in $\mathbb{F}$, but the intended semantics is an
integer or fixed-point computation. We therefore fix one public bound
$\mathcal{B}$ such that every scalar appearing in any checked identity admits an
integer interpretation in the interval $[-\mathcal{B},\mathcal{B}]$ after the
agreed fixed-point scaling.
We require that $\mathcal{B}$ is small enough to prevent wraparound in all the checked identities, and we also require that $\mathcal{B} < |\mathbb{F}|/2$ to ensure that the integer interpretation is unambiguous.

\subsection{Formal Statement Being Proved}

The public statement consists of the training dimensions
$(M,d,B,K,T)$, the learning rate $\mu$, the public tree-shape policy, the
quantization parameters, the public lookup tables used by imported
subprotocols, and the KZG setup parameters. The secret witness consists of the
quantized dataset $\mathbf{D}$, the labels $\mathbf{Y}$, the committed forest
metadata, the round-indexed boosting-state tensors, the histogram tensors, the
split metadata, the leaf values, and the auxiliary witnesses required by the
imported lookup, aggregation, softmax, and max-selection gadgets.

The relation checked by the verifier is:
there exists a committed witness encoding a valid execution trace of the fixed,
quantized trainer semantics defined in this section, together with a committed
forest and leaf values consistent with that trace.

Equivalently, an accepting proof certifies that the committed witness can be
interpreted as one end-to-end execution of the quantized training trace defined
by the paper, not as an arbitrary real-valued GBDT trainer.

\begin{theorem}[Informal security statement]
\label{thm:main-informal}
Assume the KZG commitment scheme is binding, the imported subprotocols satisfy
their claimed completeness and soundness properties, and the bounds of
\cref{prop:no-wraparound} hold. Then the protocol of Sections~4--5 satisfies:
\begin{enumerate}
  \item \textbf{Completeness.} Any honestly generated witness for the quantized
  trainer semantics is accepted.
  \item \textbf{Soundness.} Any accepting proof implies the existence of a
  witness corresponding to a valid execution of that same quantized trainer
  semantics, except with the accumulated soundness error of the imported
  gadgets and the random linear-combination checks.
  \item \textbf{Witness privacy.} If the hiding variant of the commitment scheme
  is used and the imported subprotocols are zero knowledge, then the proof
  reveals nothing beyond the public statement and the committed outputs.
\end{enumerate}
We claim soundness for this protocol, not a separate argument-of-knowledge
statement.
\end{theorem}

\subsection{What Must Be Proved}

At a high level, a proof of GBDT training must justify four kinds of steps:
\begin{enumerate}
  \item the forest is encoded correctly, and inference with the committed forest
  is correct;
  \item each boosting round computes the right pseudo-residuals from the current
  score matrix;
  \item for every node, feature, and threshold, the prover correctly derives the
  statistics needed to evaluate the split objective;
  \item the split committed in the tree is an optimizer of that objective.
\end{enumerate}

Our construction uses standard lookup arguments when rows or table values need to be
fetched \cite{cq22,cq++24,baloo25}. The new technical ingredients appear later,
when we must align heterogeneous domains, aggregate sample contributions into
histograms, and batch many polynomial identities into a small number of
openings.

Section~5 will formalize this roadmap as a sequence of propositions: forest
well-formedness, leaf assignment, leaf histogram construction, internal
propagation, prefix and total statistics, split-score correctness, split
optimality, leaf-value correctness, and finally one round-level theorem that
packages them together.
